# Git commands learning modified from remote.

## Code written with git are avilable in three spaces
- **Working directory** - The local playground where you do your development, try, fix, delete code before adding to staging.
- **Staging** - This is a virtual space where you put your code before commit. A git add *<file_name>* or ./-a command is used for staging the file/all files.
- **Git directory** - This is SHA1 hashed version of code which will be tracked by git.

> Git status helps in keeping track of what is in working, directory, staging area and git directory.
